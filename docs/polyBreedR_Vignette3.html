polyBreedR Vignette 3: Introduction to Genomic Selection

Introduction

Genomic selection (GS) leverages historical phenotype data from a breeding program to more accurately predict the genetic value of related individuals. The most important application of GS is the prediction of additive value, so that superior parents can be selected more quickly than using phenotypes alone. This is because reducing the length of the breeding cycle (not to be confused with the time to variety release) is one of the most powerful drivers of genetic gain (Cobb et al. 2019).

Historical data are organized as distinct trials that occur at a particular place and time, which are called environments in plant breeding. Proper biometric procedures are needed to combine all of this information while accounting for micro- and macro-environmental variation. The polyBreedR package uses a fully efficient, two-stage procedure (Damesa et al. 2017) that requires some entries to be replicated in each environment. In Stage 1, each environment is analyzed separately, taking into account the experimental design and (if desired) any spatial trends. Best linear unbiased estimates (BLUEs) of the genotypic (i.e., clonal) value of the individuals are computed, as well as the variance-covariance (vcov) matrix of the BLUEs. Both pieces of information are used in Stage 2 to partition the variance of the BLUEs into additive and non-additive genetic effects and genotype x environment interactions.

Phenotype Data

The tutorial dataset is based on six years (2015-2020) of potato breeding trials at the Hancock Research Station of the University of Wisconsin (UW), which is an updated version of the dataset published by Endelman et al. (2018). A mixture of augmented and partially replicated designs have been used with incomplete blocks. Four traits are included:

  • total yield (\(Mg\) \(ha^{-1}\))

  • vine maturity (1=early to 9=late)

  • specific gravity

  • fry color, measured in units of Hunter Lightness (\(L\)) after 6 months of storage

The data file also contains the stand count for each plot (out of 15 total plants), which is included as a covariate in the Stage 1 model.

Genotype Data

Our genotyping platform for tetraploid potato has been an Infinium SNP array. Most of the clones in this dataset were genotyped with Version 3 (V3) of the array, but some were genotyped with V2, and some have no marker data for various reasons. (See Vignette 1 for more information about genotype calling with polyBreedR.) When two groups of individuals have been genotyped on different platforms with many shared markers, the polyBreedR function merge_impute can be used to merge the datasets and impute the missing values by BLUP. The algorithm in merge_impute is a modification of the EM algorithm in function A.mat of the rrBLUP package, but the polyBreedR implementation is not iterative and is not recommended for use with RADseq/GBS data.

Two-stage analysis

polyBreedR functions Stage1 and Stage2 implement the two-stage analysis methodology described earlier. Both require a license for ASReml-R (version 4.1.0.148 or later), which can be obtained from VSN International. The Stage1 command expects columns named “id” and “env” in the input data frame, as well as columns for each trait. The data for each environment x trait combination are analyzed independently with a linear mixed model. Any additional fixed or random effects in the model are specified using “effects”, which is a named list of 2-vectors as shown in the example below. For each effect, the first element must be “fixed” or “random”, and the second element is “factor” or “numeric” (i.e., covariate). Stage1 returns a list with three elements:

  • “H2” is a matrix of the plot-based, broad-sense heritability for each env x trait combination

  • “blue” is a data frame of BLUEs for each individual x trait combination

  • “blue.vcov” is a list of matrices (one for each trait) with the information needed for Stage 2

Clones without marker data need to be removed from the Stage 1 results before continuing to Stage 2, which is one of the features of Stage2_prep. This function also returns a matrix called Omega, which must be saved as a variable in the global environment for ASReml-R to work properly in Stage2. The same is true for the G matrix (created above) and other covariance matrices, which are called “kernels” in Stage2. The Stage2 function automatically includes a random genetic effect with an identity (I) variance-covariance matrix, which represents non-additive effects when used in combination with a random effect with covariance proportional to G. At present, each trait must be analyzed separately in Stage 2, but a multivariate analysis to estimate the genetic correlation between traits is under development. Let’s analyze total yield.

missing.id <- setdiff(ans1$blue$id,rownames(G))
ans1b <- Stage2_prep(ans1$blue.vcov["total.yield"],exclude.id=missing.id)
Omega <- ans1b$Omega
blue <- ans1b$blue
ans2 <- Stage2(data=blue,kernels="G",silent=F)

The output “vc” contains the estimated variance components, and the results for “kernel=G” and “kernel=I” correspond to additive and non-additive genetic effects. respectively. In this example, GxE refers to the genotype x year interaction because the data come from a single location. (A tutorial for multi-location datasets is under development.) To compute the proportion of variance, the variance component for G (and other non-identity) kernels must be multiplied by the mean of the diagonal elements.

tmp <- ans2$vc[,1]*c(1,mean(diag(G)),1,1)
names(tmp) <- rownames(ans2$vc)
round(tmp/sum(tmp),2)
## kernel=I kernel=G      GxE residual 
##     0.23     0.31     0.24     0.23

Genomic narrow-sense heritability, defined here as the proportion of variance for the additive effects, is estimated at 0.31 for total yield (on a plot basis).

The Stage1 function requires partial replication within each environment to estimate the variance-covariance matrix of the BLUEs. If your dataset does not satisfy this requirement, there is the option of running Stage2 without the Omega matrix, which is suboptimal because the micro- and macro-environmental variation are confounded. The benefit of including Omega in the analysis is reflected in a lower AIC compared to a model without it.

remove("Omega") #remove variable from the global environment
ans2b <- Stage2(data=blue,kernels="G",silent=T)

#Model with Omega has lower AIC:
ans2$aic - ans2b$aic
## [1] -74.7

Genomic prediction by BLUP

One additional function, predict_MME, is needed to predict the breeding values in polyBreedR. As the name suggests, it is based on solving the Mixed Model Equations (MME). The necessary data for predict_MME are encapsulated in an S4 class called MME, which is returned by Stage2 (but can also be created explicitly by advanced users). Inversion of the MME coefficient matrix enables computation of the reliability (\(r^2\)) of the BLUPs, which is the expected squared correlation between the true and predicted values. This quantity is analogus to narrow-sense \(h^2\) and is an alternative to cross-validation (CV) for assessing genomic prediction accuracy. Cross-validation tends to underestimate prediction accuracy because the data used for validation are not true breeding values (to compensate, the correlation can be divided by narrow-sense \(h\)). On the other hand, reliability from the MME tends to overestimate accuracy because it assumes the model is correct.

The argument “weights” of predict_MME allows the user to specify what linear combination of genetic effects to predict. For parent selection based on additive values, use weights of 0 and 1 for the I and G kernels, respectively. To select on clonal values for variety release, both kernels should have weights of 1.

add.pred <- predict_MME(data=ans2$MME,weights=c(I=0,G=1))
head(add.pred[order(add.pred$blup,decreasing=T),]) #Top yielders

Previous studies have shown that the reliability of genomic prediction depends largely on the number of close relationships for each clone, which can be quantified by the 95th (or other suitably high) percentile of its G coefficients.

tab <- table(pheno$id)
Gtmp <- G
diag(Gtmp) <- NA
data2 <- data.frame(id=rownames(G),
                    G95=apply(Gtmp,1,quantile,probs=0.95,na.rm=T),
                    plots=as.integer(tab[rownames(G)]))

combo <- merge(data2,add.pred)
ggplot(combo,aes(x=G95,y=r2,colour=plots)) + geom_point() +
  scale_y_continuous("Reliability",limits=c(0.4,1),breaks=seq(0.4,1,by=0.2),labels=seq(0.4,1,by=0.2))

Because these are marker-assisted predictions, meaning they also utilize phenotypic data for the selection candidates, the number of phenotype measurements (i.e., plots) for each clone also contributes to accuracy (see above figure). The average reliability of the predicted additive values for FY3 entries, based on a single plot, is 0.62 vs. only 0.31 for the narrow-sense \(h^2\).

#Average reliability based on one plot
mean(combo$r2[combo$plots==1])
## [1] 0.619801

polyBreedR has two ways to make marker-based predictions, i.e., for individuals without phenotype data. In a real situation (as opposed to cross-validation), the G matrix will contain individuals for which BLUEs are not available in the data frame provided to Stage2, and predictions for these individuals will be present in the output from predict_MME. For cross-validation exercises, it is more efficient to mask individuals with the argument “exclude.id” in the call to predict_MME, rather than re-estimating variances for each partition of the dataset using Stage2.

To illustrate, we will mask phenotypes for the clones in the 2020 preliminary yield trial (Field Year 3), which all start with the prefix “W17”.

W17 <- blue$id[grep("W17",blue$id,fixed=T)]
add.pred.W17 <- predict_MME(data=ans2$MME,weights=c(I=0,G=1),exclude.id=W17)

combo <- merge(add.pred.W17,add.pred,by="id")[,c("id","r2.x","r2.y")]
combo$group <- ifelse(combo$id %in% W17,"W17","Remainder") 
ggplot(combo,aes(x=r2.x,y=r2.y,colour=group)) + geom_point() + geom_abline(intercept=0,slope=1) + 
  xlab(expression(r^2~"(-W17 phenotypes)")) + ylab(expression(r^2~"(+W17 phenotypes)")) + scale_colour_brewer(palette="Set1")

The above figure compares the reliability of the predicted additive values with (y-axis) vs. without (x-axis) phenotypes for the W17 group (black line = slope of 1). The reliability is substantially higher for the W17 clones when their phenotype data are included, which reinforces the difference between marker-assisted and marker-based predictions. For the rest of the population, there is a slight advantage to including the W17 phenotypes because they increase the training population size.

To estimate accuracy by cross-validation, the correlation between the predicted and observed values for the W17 group is divided by the square-root of the narrow-sense \(h^2\). The result of 0.42 is lower than the square-root of the mean reliability = 0.72, and the true value may be somewhere in between.

#Predictive ability
pred.ability <- cor(blue$blue[match(W17,blue$id)],add.pred.W17$blup[match(W17,add.pred.W17$id)])

#Prediction accuracy by cross-validation
pred.ability/sqrt(0.31)
## [1] 0.4167253
#Compared to square root of mean reliability
sqrt(mean(add.pred.W17$r2[match(W17,add.pred.W17$id)]))
## [1] 0.7237604
